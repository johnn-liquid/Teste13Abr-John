# Glossary

Created: 2 de abril de 2026 11:25

| Termo | Definição |
| --- | --- |
| **Conector / Connector** | Integração com fonte de dados externa. Não executa lógica, apenas acessa e retorna dados. |
| **Skill** | Instrução específica que um agente segue para executar uma tarefa. |
| **Agent** | Entidade que executa ações usando skills e conectores. 4 tipos: Entity, Skill, Orchestrator, Liquidity. |
| **Orquestrador / Orchestrator** | Agente maestro que coordena os demais agentes e skills. |
| **Entidade / Entity** | Abstração estruturada de objetos de mercado (SPE, Comprador, Carteira). |
| **Knowledge Base (KB)** | Repositório de contexto do usuário como arquivos, dados, informações inputadas. |
| **Artefato / Artifact** | Output gerado pelo agente: dashboards, cards, gráficos, documentos. |
| **Anexo / Attachment** | Arquivo subido pelo usuário (diferente de artefato). |
| **Task** | Uma conversa/tarefa iniciada pelo usuário com o agente. |
| **Slot** | Ponto de extensão da plataforma onde módulos (skills, conectores) plugam. |
| **Liquidez / Liquidity** | Qualquer operação em que a Liquid cobra taxa sobre o valor transacionado. |
| **Workspace** | Espaço de trabalho de uma organização, similar ao Figma/Notion. |
