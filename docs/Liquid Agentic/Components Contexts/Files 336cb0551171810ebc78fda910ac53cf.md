# Files

Atualizado em: 2 de abril de 2026
Group: Chat/Interação
Status: Draft

## O que é

Componente de visualização e gerenciamento de arquivos dentro do chat ou da Knowledge Base. Diferente de Attachments (envio) e de Artifact-Document (output gerado), Files é a representação persistente de documentos no sistema.

## Por que existe

O usuário precisa acessar, referenciar e gerenciar documentos ao longo do tempo — não apenas no momento do upload. Files é o componente que dá permanência aos documentos enviados.

## Como existe

**Exibição:**

- Lista ou grid de arquivos com nome, tipo, data e origem
- Preview inline para PDFs e imagens
- Indicador de status (indexado, em processamento, com erro)

**Ações disponíveis:**

- Abrir / visualizar
- Adicionar à Knowledge Base permanentemente
- Remover da sessão
- Referenciar em nova conversa

**Metadados exibidos:**

- Nome do arquivo, tipo, data de upload
- Quem enviou (em fluxos multiplayer)
- Se está ativo no contexto atual

## Onde é usado

- No Chat, como visualização de Attachments já enviados
- Na seção Knowledge Base da Navbar (Files + Docs)

## Quem usa

Usuário final para gerenciar documentos. Skills para referenciar conteúdo.
