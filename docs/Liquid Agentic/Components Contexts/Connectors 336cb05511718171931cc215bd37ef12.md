# Connectors

Atualizado em: 2 de abril de 2026
Group: Sistema
Status: Draft

## O que é

Integrações com fontes externas de dados — birôs de crédito, APIs regulatórias, Open Finance, sistemas de cartório, CRIs/CRAs e outros provedores. O produto não define quais conectores existem: ele cria o **processo** para que novos conectores sejam descobertos, validados e ativados continuamente.

## Por que existe

As Skills dependem de dados externos para funcionar. Um conector é o canal certificado e auditável entre o sistema e uma fonte de dados — com responsabilidade legal pela Liquid sobre cada consulta.

## Como existe

**No chat (indicador):**

- Quando um conector é consultado durante uma skill, o agente indica quais fontes foram utilizadas
- Ex: "Consulta realizada via Serasa Experian" com timestamp e referência auditável
- Clicável para ver detalhes da consulta

**No painel de gerenciamento (Navbar → Conectores):**

- Lista de conectores ativos/inativos disponíveis para a conta
- Status de cada conector: ativo, em aprovação, com erro de autenticação
- Botão "Create new connector"
- Cada conector tem: nome, fonte, tipo de dado, data de ativação, permissões

**Exemplos BR:** Serasa, Boa Vista, PH3A, CRI/CRA, cartório, Open Finance

**Exemplos globais:** bureaus locais por país

**Regras:**

- Consultas via birô usam credenciais da Liquid com responsabilidade legal
- Todo uso de conector é logado para auditoria
- O sistema é agnóstico ao conector — nova skill pode usar qualquer conector ativo

## Onde é usado

- Chat: indicador inline nas respostas das Skills
- Navbar: seção de gerenciamento de conectores

## Quem usa

- Sistema (Skills invocam conectores automaticamente)
- Admin Liquid (aprova e gerencia novos conectores)
- Usuário avançado (pode ver quais conectores estão ativos)
