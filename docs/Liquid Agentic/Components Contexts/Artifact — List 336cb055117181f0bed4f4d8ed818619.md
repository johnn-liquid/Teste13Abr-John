# Artifact — List

Atualizado em: 6 de abril de 2026
Group: Artefatos
Status: Draft

## O que é

Artefato de listagem estruturada. Apresenta conjuntos de itens ordenados ou categorizados — resultados de busca, rankings, sequências de etapas, comparações.

## Por que existe

Muitos outputs de skill são naturalmente listas: empreendimentos encontrados, documentos necessários, etapas de um processo, itens de checklist. O artefato list dá estrutura e escaneabilidade a esses conteúdos.

## Como existe

**Tipos:**

- `ordered` — lista numerada (etapas, ranking, prioridades)
- `unordered` — lista de itens sem ordem específica
- `checklist` — itens com estado de conclusão (✓ / pendente)
- `grouped` — itens agrupados por categoria

**Comportamento:**

- Itens podem ser expansíveis (accordion) para revelar detalhes (reduz carga cognitiva progressivamente)
- Ações por item: selecionar, abrir, exportar linha
- Pode ser filtrada e ordenada pelo usuário
- Em checklists: itens marcáveis pelo usuário (ex: documentos coletados)

**Comportamento de Escalabilidade e Densidade:**

- **Scroll Infinito vs. Paginação:** Listas devem usar *scroll infinito* (lazy loading) e manter o fluxo de leitura ininterrupto. Nunca use paginação em listas para evitar atrito cognitivo.
- **Densidade (Spacious vs. Compact):**
    - `compact`: usado quando a meta primária é **escaneabilidade rápida** (ex: checklists curtos, resultados de busca estreitos no chat).
    - `spacious`: usado quando o foco é **leitura atenta** e tomada de decisão sobre o item (aumentando a área de respiro e diminuindo o ruído visual).

### Regra de quando usar (Lista vs. Tabela)

- **Limite de Atributos:** Se cada item possui **até 3 atributos principais**, use Lista. O cérebro humano processa melhor formatos lineares até este limite (Lei de Miller).
- **Comparabilidade:** Se o usuário precisa comparar atributos lado-a-lado ou ordenar por múltiplas métricas cruzadas, abandone a lista e **promova para Datatable**.
- **Volume Seguro:** Listas funcionam bem como sumários (< 10 itens visíveis na tela). Para dezenas de itens puramente analíticos, priorize Tabela.

**Exemplos de uso:**

- Lista de documentos necessários para uma operação
- Empreendimentos que atendem a critérios de busca
- Etapas pendentes no processo de aprovação
- Pendências identificadas em uma análise

## Onde é usado

Chat (inline para listas curtas) e painel split para listas mais extensas.

## Quem usa

Gerado pelos Skill Agents. Consumido pelo usuário para acompanhar processos ou tomar decisões.
