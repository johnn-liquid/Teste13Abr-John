# User Input

Atualizado em: 2 de abril de 2026
Group: Chat/Interação
Status: Draft

## O que é

Campo de entrada onde o usuário digita mensagens, envia comandos ou fornece dados para o sistema. É a principal forma de comunicação do usuário com o Orchestrator.

## Por que existe

É a porta de entrada do produto. Toda intenção do usuário passa por aqui — o Orchestrator lê o input para identificar qual skill acionar, qual conector consultar e qual contexto é relevante.

## Como existe

**Modos:**

- `text` — input livre, linguagem natural
- `structured` — formulário contextual gerado pelo agente (ver Inputs Estruturados)
- `voice` — entrada por voz (ver Voice)

**Comportamento:**

- Suporta texto livre por padrão
- Pode ser transformado em input estruturado quando o Orchestrator identifica que a tarefa exige dados específicos
- Aceita anexos via drag-and-drop ou botão (ver Attachments)
- Suporte a menções (`@conector`, `@skill`, `@entidade`) para invocação explícita
- Expansível verticalmente para inputs longos
- Ação de envio: `Enter` para enviar, `Shift+Enter` para quebra de linha

**Estados:**

- `empty` — placeholder contextual
- `typing` — usuário digitando
- `processing` — aguardando resposta do sistema (desabilitado)
- `error` — falha no envio ou input inválido

## Onde é usado

Fixo na base do Chat. Presente em todos os layouts.

## Quem usa

Usuário final. O Orchestrator processa o output deste componente.
