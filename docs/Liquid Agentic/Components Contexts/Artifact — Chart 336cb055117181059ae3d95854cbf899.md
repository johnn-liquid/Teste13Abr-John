# Artifact — Chart

Atualizado em: 6 de abril de 2026
Group: Artefatos
Status: Draft

## O que é

Artefato de visualização de dados quantitativos — séries históricas, comparativos de mercado, distribuições, tendências. Gerado pelos Entity Agents e Skill Agents a partir de dados de conectores.

## Por que existe

Dados do mercado imobiliário e financeiro têm valor visual. Um gráfico de evolução de preço por m² ou de inadimplência por região comunica mais rápido que uma tabela — e justifica decisões de forma mais convincente para o cliente.

## Como existe

**Tipos:**

| Tipo | Quando usar | Exemplo de uso |
| --- | --- | --- |
| `line` | Séries temporais com tendência contínua | Evolução do preço/m², índices de correção, taxas de juros |
| `bar` | Comparativos entre categorias discretas | Volume por região, aprovações por banco, performance por período |
| `area` | Volumes acumulados ou composição ao longo do tempo | Carteira acumulada, fluxo de pagamentos projetado |
| `donut` / `pie` | Distribuição proporcional de um todo | Mix de carteira por tipo, distribuição de risco por faixa |
| `heatmap` | Intensidade em duas dimensões | Inadimplência por região × período, demanda por tipo × bairro |
| `gauge` | Score pontual em escala visual | Score de crédito, score de liquidez, rating de ativo |
| `scatter` | Correlação entre duas variáveis | Renda vs. valor do imóvel, prazo vs. taxa |

**Comportamento:**

- Interativo: hover para detalhe, zoom, filtros de período
- Alimentado por dados em tempo real quando gerado por Entity Agent
- Estático quando gerado por Skill a partir de consulta pontual
- Exportável como PNG ou CSV (dados brutos)
- Título e legenda sempre presentes, fontes indicadas

### Variantes de tamanho e estado

**Tamanhos:**

- `inline` — mini-chart dentro de card ou KPI-sparkline (sem eixos, sem legenda)
- `default` — painel split (eixos e legenda compactos)
- `full` — dashboard dominante (todos os controles visíveis)

**Estados:**

- `loading` — skeleton animado na área do gráfico
- `empty` — sem dados para o período selecionado (mensagem contextual)
- `error` — falha no conector (ícone + mensagem)
- `stale` — dados fora do intervalo esperado de atualização
- `fallback` — quando o agente gera um formato não suportado, reverte automaticamente para `datatable` para preservar o acesso à informação bruta sem quebrar a interface (princípio de mitigação de erro)

### Tokens usados

**Cor:**

- Série primária: `color-chart-primary`
- Séries adicionais: `color-chart-2` ... `color-chart-6` (paleta sequencial)
- Positivo/crescimento: `color-feedback-success`
- Negativo/queda: `color-feedback-error`
- Neutro/referência: `color-chart-neutral`
- Grid/eixos: `color-border-subtle`
- Background: `color-surface-default`

**Tipografia:**

- Título do gráfico: `type-label-md` / semibold
- Labels de eixo: `type-label-xs` / regular
- Tooltip: `type-body-sm` / regular
- Legenda: `type-label-sm` / regular

**Espaçamento:**

- Padding da área do gráfico: `space-16`
- Gap título → gráfico: `space-12`
- Gap gráfico → legenda: `space-16`

### Regra de quando usar

- Use `line` para qualquer dado com dimensão temporal — nunca `bar` para séries contínuas
- Use `bar` quando a comparação entre categorias é mais importante que a tendência
- Use `donut` para composição proporcional com até 6 categorias — acima disso, use `bar` horizontal
- Use `heatmap` quando há duas dimensões categóricas e a intensidade é o dado relevante
- Use `gauge` apenas para scores com escala definida (0–1000, 0–100%) — não para valores absolutos
- Nunca use `pie` — substitua sempre por `donut` ou `bar` horizontal para melhor legibilidade

## Onde é usado

Painel de mercado (dashboard dominante). Chat split quando parte de uma análise. Inline no chat para respostas rápidas com dado visual.

## Quem usa

Gerado pelos Entity Agents (mercado) e Skill Agents (análises). Consumido pelo analista e pelo usuário do painel de mercado.
