# Skills

Group: Sistema
Status: Draft

## O que é

Instruções determinísticas invocadas pelo Orchestrator para executar uma tarefa específica. Cada skill é um Skill Agent — especializado, com input e output definidos, sem variação para a mesma entrada. O produto não define quais skills existem: ele cria o **processo** para que novas skills sejam geradas e validadas continuamente.

## Por que existe

Em mercado regulado, o agente não pode "achar" uma resposta — ele precisa executar um processo auditável. Skills são a camada que garante determinismo: underwriting, análise de crédito, pricing e análise de documento sempre retornam resultados consistentes para os mesmos inputs.

## Como existe

**No chat (execução):**

- Invocada pelo Orchestrator com base na intenção do usuário
- Invisível para o usuário — ele vê o resultado (artefato), não a skill em si
- Pode ser invocada explicitamente com `@skill-name`
- Indicador visual durante processamento

**No painel de gerenciamento (Navbar → Skills):**

- Lista de skills disponíveis para a conta
- Status: ativa, em aprovação, em testes
- Botão "Create new skill"
- Cada skill tem: nome, descrição, conectores necessários, tipo de output, versão

**Exemplos:** underwriting, análise de crédito, scoring, pricing, análise de documento, PDD, viabilidade de empreendimento, deep research de mercado

**Regras:**

- Skills são determinísticas — mesma entrada, mesmo resultado
- Skills da Liquid não podem alucinar
- Cada execução é logada com input, output e versão da skill

## Onde é usado

- Chat: executadas em background, output como artefato
- Navbar: seção de gerenciamento de skills

## Quem usa

- Invocadas pelo Orchestrator
- Gerenciadas pelo Admin Liquid
- Visíveis (resultado) para o usuário final
