# Chat

Atualizado em: 2 de abril de 2026
Group: Chat/Interação
Status: Draft

## O que é

O container principal de interação entre o usuário e o sistema. É o espaço onde a conversa acontece — mensagens, respostas do agente, artefatos gerados e ações sugeridas coexistem em uma linha do tempo.

## Por que existe

É o ponto de contato central do produto. Todo o valor do sistema — análises, dados, skills, conectores — se manifesta aqui. O chat não é um recurso: é o palco.

## Como existe

Três modos de layout, determinados pelo contexto da interação:

| Layout | Quando | Proporção |
| --- | --- | --- |
| **Chat central** | Conversa simples, discovery, perguntas abertas | 100% chat |
| **Split 50/50** | Análise com ida e volta (ex: análise de crédito) | 50% chat / 50% artefato |
| **Dashboard dominante** | Visualização de relatório, painel de dados | Chat minimizado |

**Estados:**

- `idle` — aguardando input do usuário
- `processing` — orchestrator roteando, skill sendo executada
- `streaming` — resposta sendo gerada em tempo real
- `artifact-generated` — artefato pronto, layout pode expandir

**Regras:**

- O histórico da conversa é persistido por sessão
- Sessões multiplayer existem dentro do chat — o sistema sabe quem está falando
- O layout muda dinamicamente conforme o tipo de output gerado

## Onde é usado

Tela principal do produto. É o componente raiz da experiência.

## Quem usa

Todos os perfis (corretor, analista, comprador). O Orchestrator também "vive" aqui — coordena o fluxo invisível que acontece em resposta ao input do usuário.
