# Attachments

Atualizado em: 2 de abril de 2026
Group: Chat/Interação
Status: Draft

## O que é

Arquivos enviados pelo usuário para o sistema durante uma conversa. São inputs — não outputs. A distinção com Artefatos é fundamental: anexo é o que o usuário **traz**; artefato é o que o agente **gera**.

## Por que existe

O sistema precisa processar documentos reais do cliente: contratos, extratos, laudos, documentos de identidade.

## Como existe

**Formatos suportados:**

- PDF, DOCX, XLSX — documentos financeiros e contratuais
- PNG, JPG — documentos de identidade, laudos fotográficos
- CSV — dados tabulares de carteiras

**Comportamento:**

- Upload via drag-and-drop no chat ou botão no User Input
- Preview inline no chat após upload
- O sistema identifica automaticamente o tipo de documento e sugere a skill relevante
- Pode ser adicionado à Knowledge Base permanentemente ou usado apenas na sessão atual

**Estados:**

- `uploading` — transferência em andamento
- `processing` — agente lendo e indexando
- `ready` — disponível para referência
- `error` — formato inválido ou falha no upload

## Onde é usado

No Chat, integrado ao User Input. Aparece como preview no fluxo da conversa.

## Quem usa

Enviado pelo usuário. Processado pelas Skills e pelo Knowledge Base.
