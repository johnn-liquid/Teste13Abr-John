# Artifact — Link

Atualizado em: 7 de abril de 2026
Group: Artefatos
Status: Draft

## O que é

Referência navegável gerada pelo agente — URL externa, deep link interno ou sugestão de próximo passo. É o artefato de menor densidade informacional: o valor está no destino, não no conteúdo gerado.

## Por que existe

Nem toda resposta útil é uma análise. Em muitos momentos o agente deve apontar o caminho — para uma simulação bancária, para um relatório completo, para o próximo passo no funil. O Link formaliza essa navegação sem criar um artefato de conteúdo desnecessário.

## Como existe

### Tipos

| Tipo | Quando usar |
| --- | --- |
| `link-cta` | Ação direta de navegação para o próximo passo do funil. Ex: "Abrir simulação bancária", "Iniciar operação em Liquidity Ops" |
| `link-card` | Destino externo que precisa de contexto para o usuário decidir se vai clicar. Exibe título + descrição + domínio (estilo unfurl) |
| `link-suggestion` | Camada 1 — sugestão de próximo tópico de conversa, SEO-indexável, dispara um prompt pré-definido |
| `link-inline` | Referência dentro do texto do agente, sem destaque visual próprio. Não é um artefato isolado |

### Regra de quando usar

Use Link quando **o valor para o usuário está no destino**, não no conteúdo que o agente gera.

**Link vs. Card:** Card tem conteúdo próprio que informa uma decisão. Se o agente precisa dizer algo antes da ação, é Card com CTA no footer — não Link. Se a navegação *é* o artefato completo, é Link.

**Link vs. Suggestion:** Suggestion dispara uma nova mensagem no chat. Link navega para fora do chat ou para outra área do produto.

**Não use** `link-cta` quando o usuário precisa de mais contexto antes de agir — use Card.

**Não use** Link para comunicar um resultado de análise. Qualquer output com dado ou interpretação é outro tipo de artefato.

`link-inline` nunca aparece como artefato isolado — existe apenas dentro do texto de uma mensagem do agente.

## Onde é usado

`link-cta` e `link-card` aparecem no chat ou no painel split como artefatos isolados. `link-suggestion` aparece na área de sugestões da interface, abaixo do input. `link-inline` aparece dentro do texto do agente.

## Quem usa

Gerado pelo Orchestrator (navegação de funil e próximos passos) e pelos Skill Agents (referências a fontes externas ou documentos relacionados). `link-suggestion` é gerado pelo Orchestrator com base no contexto da sessão e no perfil da Camada 1.
