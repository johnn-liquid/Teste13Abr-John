# Artifact — Card

Atualizado em: 6 de abril de 2026
Group: Artefatos
Status: Draft

## O que é

Artefato compacto que apresenta um resultado resumido de forma visual e escaneável. É o tipo de artefato mais frequente em respostas rápidas — aprovações, perfis de entidade, scores.

## Por que existe

Nem todo output precisa de um relatório completo. O card entrega o essencial de forma imediata — ideal para o fluxo de aprovação do MVP, onde o corretor precisa de uma resposta clara e rápida.

## Como existe

**Estrutura típica:**

- Header: título + status (aprovado / pendente / atenção)
- Corpo: 3–6 atributos-chave com valor e indicador visual
- Footer: fonte, data, ações disponíveis orientadas por forte hierarquia visual.
    - **Ação Primária (alta saliência visual):** o "caminho feliz" preditivo (ex: "Start simulation"). Botão preenchido (`filled`).
    - **Ação Secundária (baixa saliência visual):** rotas alternativas ou informativas (ex: "View full report"). Botão em contorno ou texto (`outline`/`ghost`) para não competir atencionalmente e evitar fadiga de decisão.

**Variações:**

| Tipo | Conteúdo típico | Skill que gera |
| --- | --- | --- |
| `approval-card` | Resultado de análise de crédito: score, renda compatível, banco elegível, próximo passo | Credit analysis skill |
| `entity-card` | Perfil de comprador, SPE ou empreendimento: dados cadastrais, histórico, indicadores-chave | Entity lookup skill |
| `liquidity-card` | Oportunidade de liquidez identificada: ativo, score de liquidez, dimensões avaliadas, CTA para Liquidity Ops | Liquidity Agent |
| `portfolio-card` | Ficha resumo de carteira: total, inadimplência, distribuição, variação | Portfolio analysis skill |
| `score-card` | Score isolado com contexto: valor, faixa, histórico recente | Scoring skill |

**Comportamento:**

- Aparece inline no chat ou no painel split
- Pode ser expandido para o artefato completo (document)
- CTA no footer leva ao próximo passo do funil
- Em aprovação: linguagem sempre orientada à possibilidade, não à negação

### Variantes de tamanho e estado

**Tamanhos:**

- `compact` — 1–3 atributos, usado inline no chat
- `default` — 3–6 atributos, padrão no split
- `expanded` — pré-visualização antes de abrir o document completo

**Estados:**

- `loading` — skeleton de card
- `success` — análise positiva (borda ou badge verde)
- `warning` — análise com ressalvas (amarelo)
- `pending` — aguardando dados ou ação do usuário (neutro)

### Tokens usados

**Cor:**

- Borda/badge success: `color-feedback-success`
- Borda/badge warning: `color-feedback-warning`
- Borda/badge pending: `color-border-default`
- Background do card: `color-surface-raised`
- Título: `color-text-primary`
- Atributos secundários: `color-text-secondary`

**Tipografia:**

- Título do card: `type-label-md` / semibold
- Valor de atributo: `type-body-md` / medium
- Rótulo de atributo: `type-label-sm` / regular
- Status badge: `type-label-xs` / semibold uppercase

**Espaçamento:**

- Padding do card: `space-16`
- Gap entre atributos: `space-12`
- Gap header → corpo: `space-16`
- Gap corpo → footer: `space-12`

### Regra de quando usar

- Use card quando o resultado pode ser consumido e decidido sem leitura aprofundada
- Use `approval-card` para qualquer saída de análise de crédito — nunca texto puro
- Use `entity-card` sempre que uma entidade (CPF, CNPJ, empreendimento) é o objeto central da resposta
- Use `liquidity-card` apenas quando o Liquidity Agent identificar oportunidade — não para análises genéricas
- Se o resultado exige mais de 6 atributos ou narrativa, promova para `artifact-document`

## Onde é usado

Chat (inline) e painel split. Frequente no MVP do corretor.

## Quem usa

Gerado pelos Skill Agents. Lido pelo usuário final para tomada de decisão imediata.
